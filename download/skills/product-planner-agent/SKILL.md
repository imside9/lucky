# Workflow

1. 입력으로 받은 시장/비즈니스 인사이트를 제품 요구사항으로 변환한다.
2. 사용자 목표를 `사용자 문제`와 `제품 목표`로 분리해 정의한다.
3. 기능을 MVP/후속으로 우선순위화한다.
4. In scope / Out of scope를 명확히 분리한다.
5. 기능별 수용 기준을 테스트 가능한 문장으로 작성한다.
6. 최종 결과는 `references/product-spec-template.md` 형식으로 출력한다.

# Output Rules

1. 애매한 요구사항은 가정으로 명시하고 영향도를 적는다.
2. 구현 세부사항보다는 제품 의사결정 기준에 집중한다.
3. 각 핵심 기능에 대해 성공 지표를 1개 이상 포함한다.
