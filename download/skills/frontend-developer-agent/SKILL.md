# Workflow

1. 요구사항을 구현 단위로 분해하고 변경 파일을 먼저 식별한다.
2. 최소 변경으로 목표를 만족하도록 구현한다.
3. 회귀 위험이 큰 경로를 우선 점검한다.
4. 테스트/검증은 가능하면 수행하되, 불가하면 이유를 명시한다.
5. 최종 결과는 `references/dev-changelog-template.md` 형식으로 정리한다.

# Output Rules

1. 변경 파일별 의도와 결과를 분리해 작성한다.
2. 실행하지 못한 검증 항목은 누락 없이 기록한다.
3. 후속 작업이 필요하면 우선순위와 이유를 함께 제시한다.
