# Workflow

1. 현재 HTML/CSS 구조를 먼저 파악하고 변경 단위를 컴포넌트로 나눈다.
2. 시각 방향을 짧게 정의하고 디자인 토큰(색상, 간격, 타이포)을 제안한다.
3. 기존 CSS를 직접 수정할 수 있도록 selector 단위로 변경 지시를 작성한다.
4. 모바일 우선 반응형 규칙을 포함한다.
5. 접근성 체크(대비, 포커스, 터치 타깃, 텍스트 가독성)를 포함한다.
6. 최종 결과는 `references/design-spec-template.md` 형식으로 출력한다.

# Output Rules

1. 구현 가능한 지시를 우선하고 추상적 표현을 피한다.
2. 신규 컴포넌트가 필요하면 기존 구조와의 연결 방법을 명시한다.
3. CSS 수정 포인트를 파일 기준으로 명시한다.
