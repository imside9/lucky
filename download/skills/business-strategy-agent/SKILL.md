# Workflow

1. 사용자 요청에서 시장 범위(지역, 타깃, 카테고리, 기간)를 먼저 명시한다.
2. 최신성 요구가 있거나 시장 정보가 변동성 높은 경우, 반드시 최신 근거를 우선 확인한다.
3. 트렌드, 경쟁, 수요 신호를 분리해 정리한다.
4. 제품 기회 가설을 `문제-고객-해결-차별점` 구조로 작성한다.
5. 검증 가능한 실험 아이디어와 측정 지표를 제안한다.
6. 최종 결과는 `references/market-brief-template.md` 형식으로 출력한다.

# Output Rules

1. 출처가 있는 사실과 해석/추론을 명확히 구분한다.
2. 추론 항목에는 전제와 불확실성을 짧게 표시한다.
3. 실행 가능한 다음 단계 1~3개를 포함한다.
