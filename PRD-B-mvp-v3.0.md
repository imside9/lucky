# PRD 3.2: B안 30분 데모 - 하이텐션 탭 버스트 웹앱

- 버전: v3.2
- 작성일: 2026-02-13
- 목표: 30분 내 클릭 가능한 단일 페이지 MVP 완성

## 1) 제품 정의
- 한 줄: 15초 탭 러시에서 콤보를 쌓아 버튼을 점진적으로 폭발시키고, 결과를 공유하는 모바일 웹앱
- 타깃: 짧은 공백 시간에 즉시 즐길 거리를 찾는 10대 사용자
- 핵심 가치: 즉시 시작, 즉시 결과, 즉시 공유

## 2) 성공 기준
1. 사용자가 30초 내 `시작 -> 플레이 -> 결과 -> 공유 -> 재시작` 루프를 완료할 수 있다.
2. 클릭 데모가 단일 HTML/CSS/JS로 동작한다.
3. 모바일(360px/390px)에서 주요 버튼이 보이고 탭 가능하다.
4. 플레이 중 버튼 스케일업/버스트, 사운드, 하이프 텍스트가 체감된다.

## 3) MVP 범위
### In Scope
1. 3개 화면(Start/Game/Result)
2. 10~20초 타이머(기본 15초)
3. 탭 점수 증가 액션 + 콤보
4. 최고점 저장(localStorage)
5. 결과 텍스트 복사(공유 fallback 포함)
6. 탭 사운드(on/off)
7. 버튼 점진적 버스트 연출(스케일/글로우/폭발 임팩트)

### Out of Scope
1. 백엔드/DB/인증
2. 실시간 랭킹/멀티플레이
3. 신고/모더레이션
4. Cloudflare API 연동

## 4) 사용자 시나리오
1. 시작 화면에서 `게임 시작`
2. 게임 화면에서 15초 동안 탭해 점수/콤보 획득
3. 타이머 종료 후 결과 화면 전환
4. `결과 공유`로 텍스트 복사
5. `다시 하기`로 새 게임 시작

## 5) 기능 요구사항
- FR-1: 시작 버튼 클릭 시 게임 상태 시작
- FR-2: 15초 카운트다운 및 실시간 점수 표시
- FR-3: 탭 액션마다 점수 +1 및 콤보 증가
- FR-4: 탭 시 사운드 재생(설정으로 음소거 가능)
- FR-5: 콤보 구간별 버튼 시각 연출 강화
- FR-6: 종료 시 이번 점수/최고점/랭크 표시
- FR-5: 공유 버튼으로 결과 문자열 복사
- FR-6: 재시작 시 상태 초기화

## 6) 비기능 요구사항
- NFR-1: 로딩 없이 즉시 시작
- NFR-2: 주요 버튼 터치 영역 44px 이상
- NFR-3: 에러 발생 시 안내 문구 표시(복사 실패 등)

## 7) 화면 구성
1. Start: 타이틀, 설명, 시작 버튼
2. Game: 타이머, 점수, 콤보, 탭 버튼(버스트 연출), 사운드 토글
3. Result: 점수, 최고점, 공유/재시작 버튼

## 8) 데이터 모델
```js
{
  state: "idle" | "playing" | "ended",
  score: number,
  bestScore: number,
  durationSec: 15,
  timeLeft: number,
  combo: number,
  soundOn: boolean
}
```

## 9) 수용 기준
1. 타이머 0초 시 자동 결과 화면으로 이동
2. 공유 버튼 클릭 시 텍스트 복사 성공 또는 fallback 안내 노출
3. 재시작 시 점수는 0으로 초기화되고 최고점은 유지
4. 사운드 OFF에서도 오류 없이 플레이 가능
5. 콤보 구간별 버튼 시각 변화가 동작

## 10) 릴리스 기준
- Go: 핵심 루프 전부 동작, 치명 오류 없음
- No-Go: 화면 전환 불가, 타이머/점수 로직 오류, 버튼 무반응
